What is Semantic HTML?

HTML is often messy. A lot of what you see online has too many div containters and not enough headings. This makes it especially difficult for users with screen readers to access to internet.

Semantic HTML means using HTML tages related to the meaning of the structure and content of your page. Not only does this make a site accessible, but it makes your code clear and concise.


Tags

Lets go over the semantic meaning of common tags...

  • <header> — Introductory content for a page or section.
  • <footer> — Closing section containing metadata or links.
  • <h1 ... h6> — Hierarchical headings that structure content.
  • <main> — The primary, unique content of the page.
  • <nav> — Major navigational links for the site or section.
  • <section> — A thematic grouping of related content.
  • <aside> — Supplementary or tangential information.
  • <figcaption> — A caption describing a figure.

Structure

A typical structure begins with a <header> that introduces the site or page, often followed by a <nav> element containing the main navigation links. The core content is placed inside <main>, which should appear only once and contain the primary information users come for.

Within <main>, you can divide content using <section> to establish thematic areas or <article>for pieces of content that could stand alone. Headings (<h1> through <h6>) should be used in order to create a clear outline that reflects your content hierarchy. Supplementary or tangential information, such as sidebars or related links, belongs inside an <aside> so that assistive technologies can easily distinguish it from the main flow. Finally, a <footer> closes the page or section with metadata, links, or additional information.

Example of a Semantic Page Structure


<header>
    <h1>My Web Development Blog</h1>
    <nav>
        <a href="/">Home</a>
        <a href="/tutorials">Tutorials</a>
        <a href="/about">About</a>
    </nav>
</header>

<main>
    <section>
        <h2>Latest Posts</h2>

        <article>
            <h3>Understanding Semantic HTML</h3>
            <figure>
                <img src="semantic.png" alt="Semantic layout example">
                <figcaption>A diagram illustrating semantic layout.</figcaption>
            </figure>
            <p>Semantic HTML helps define the meaning of your content...</p>
        </article>
    </section>

    <aside>
        <h4>About This Blog</h4>
        <p>Tips for writing clean, accessible HTML and CSS.</p>
    </aside>
</main>

<footer>
    <p>© 2025 My Web Development Blog</p>
</footer>
